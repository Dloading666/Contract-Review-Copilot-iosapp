# 单模型方案归档

## 说明

这份文档保留为历史设计归档，用来说明项目曾讨论过“前端移除模型选择、后端收敛为单一模型”的产品方向。

当前项目不再以特定厂商模型名称作为对外展示文案，实际运行模型以系统配置为准：

- `review_model`
- `ocr_model`

## 归档结论

- 前端不应把某个具体模型名称直接写死在界面文案里
- `/api/models` 返回值应来自后端当前配置，而不是历史品牌名
- OCR 校对、合同审查、问答和报告生成的实际模型来源，应统一由配置项决定

## 后续约束

- 页面文案使用“审查模型”“文字校对”“智审内核”这类中性表达
- 示例配置文件使用当前通用配置项：
  - `OPENAI_API_KEY`
  - `OPENAI_BASE_URL`
  - `REVIEW_MODEL`
  - `OCR_MODEL`
- 测试断言应校验当前配置行为，不再写死历史模型名称
