## 目标

将未登录首页重做为与 [screen.png](/D:/agent%20project/Contract-Review-Copilot/%E5%89%8D%E7%AB%AF%E7%A4%BA%E8%8C%83%E7%95%8C%E9%9D%A2/%E9%A6%96%E9%A1%B5/screen.png) 尽量一比一一致的 React 页面，并将 [code.html](/D:/agent%20project/Contract-Review-Copilot/%E5%89%8D%E7%AB%AF%E7%A4%BA%E8%8C%83%E7%95%8C%E9%9D%A2/%E9%A6%96%E9%A1%B5/code.html) 中的交互与动画质感完整迁移到现有前端项目中。

## 用户目标

1. 用户首次进入网站时，看到与示范图高度一致的营销首页。
2. 首页整体不是静态展示，而是持续存在轻微但明确的动效。
3. 用户点击首页按钮后，仍进入现有登录/注册流程，不引入新的独立入口逻辑。
4. Docker 部署后的首页视觉和交互应与本地开发环境一致。

## 动画与交互设计

本次首页不仅需要入场动画，还需要“持续存在的轻动画”。

### 1. 按钮按压动效

完整保留 `code.html` 中 brutalist button 交互：

- 默认：`4px 4px` 硬阴影
- `hover`：向右下位移 `2px`
- `active`：完全压平到 `0px`

适用范围：

- 顶部 `登录` / `免费注册`
- Hero 主按钮
- 底部 CTA 两个按钮

### 2. 首屏持续动画

首页打开后持续存在以下轻动画：

- Doge 主视觉轻微上下浮动
- 贴纸便签轻微摆动
- `FREE` 背景字缓慢漂移
- Hero 文本区保持极轻微呼吸感，不做明显位移

要求：

- 动画周期较长，避免烦躁
- 不影响可读性
- 全程持续，不是只播放一次

### 3. 卡片持续动效

- 功能卡片保留轻微 hover 抬升
- 口碑卡保留轻微悬浮
- 技术来源面板保持稳定，不做大幅漂移，只允许极弱层次感呼吸

### 4. 首次入场动效

页面首次渲染时使用轻入场：

- Hero 左右两区分批显现
- 功能卡片依次出现
- 三步流程、技术来源、口碑区、CTA 采用分区 reveal

入场方向以淡入和轻微上移为主，不做复杂旋转或滑屏特效。

### 5. 滚动显现

当用户向下滚动时，各分区首次进入视口再触发 reveal：

- 功能卡片区
- 三步流程区
- 技术来源区
- 口碑区
- CTA 区

实现建议：

- 使用 `IntersectionObserver`
- 不引入额外动画库

### 6. 可访问性

支持 `prefers-reduced-motion`：

- 减少动画偏好开启时，关闭持续漂浮和滚动 reveal
- 保留按钮按压反馈和基础状态变化

## 风险与注意事项

1. `code.html` 中中文存在部分编码异常，实施时应以示范图和可辨识原文进行修正，不应机械照搬乱码文本。
2. 如果继续使用外链图片，Docker 部署后可能受外部资源失效影响，因此优先本地化资源。
3. 首页为长页视觉，还原时需要控制 CSS 复杂度，避免污染登录/注册等其他页面。
