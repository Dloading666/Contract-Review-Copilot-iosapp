# 免责声明入口弹窗设计

## 目标

在用户使用网站前展示一条强制确认的免责声明，明确说明平台仅提供合同风险扫描与辅助提示，不构成法律意见，且相关使用风险由用户自行判断承担。

## 方案

采用入口级全屏弹窗。应用启动时先检查浏览器本地是否已经记录同意状态；若未同意，则优先展示免责声明弹窗，要求用户勾选确认后才能进入登录页或主工作台。

## 交互规则

- 同一设备同一浏览器只需确认一次。
- 确认前按钮保持禁用，必须先勾选“我已知悉并同意上述免责声明”。
- 确认后将状态写入 `localStorage`，键名带版本号，后续如需更新文案可通过升级版本重新触发确认。

## 实现边界

- 免责声明只拦截前端使用入口，不改变登录、上传、审查、问答等业务接口。
- 当浏览器存储不可用时，本次会话内仍可继续使用，但无法跨刷新记住同意状态。

## 验证

- 未同意时显示免责声明弹窗且无法进入主界面。
- 勾选后“同意并继续”可点击。
- 同意后刷新页面，不再重复弹出。
